## 一、设计原则

### 1.1 核心原则

| 原则 | 说明 | 应用示例 |
|------|------|----------|
| **减少认知负担** | 清晰的导航，明确的操作反馈 | 每页只做一件事，操作按钮位置固定 |
| **提升操作效率** | 批量操作，快捷键支持 | 表格支持批量选择，常用操作3次点击内完成 |
| **保证数据安全** | 操作确认，版本回溯 | 删除前确认，发布前预览 |
| **促进团队协作** | 实时同步，状态可见 | 翻译状态实时更新，审核任务清晰分配 |

### 1.2 体验指标

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 首次发布完成时间 | < 5分钟 | 从登录到获取CDN链接 |
| 找到待翻译词条 | ≤ 3次点击 | 从工作台到编辑器 |
| 编辑器响应延迟 | < 200ms | 输入到显示反馈 |
| 操作成功率 | > 95% | 提交/发布等核心操作 |

## 四、核心用户流程

### 4.1 流程总览

```
┌─────────────────────────────────────────────────────────────┐
│  MVP核心流程                                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  流程1: 新用户首次体验                                       │
│  登录 → 创建项目 → 邀请成员 → 导入词条(预览) → 首次发布       │
│                                                             │
│  流程2: 译员日常工作                                         │
│  工作台 → 编辑翻译 → 提交审核                                │
│                                                             │
│  流程3: 审核员工作                                           │
│  审核工作台 → 查看对比 → 通过/退回                           │
│                                                             │
│  流程4: 发布管理员操作                                       │
│  创建发布 → 变更预览 → 门禁检查 → 确认发布 → CDN交付          │
│                                                             │
│  流程5: 项目管理                                             │
│  项目设置 → 成员管理 / 命名空间 / 语言配置                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 流程1：新用户首次体验

| 环节 | 体验标准 | 验收指标 |
|------|----------|----------|
| **登录** | 一键OAuth，无密码负担 | 登录成功率 > 95%，平均耗时 < 10秒 |
| **创建项目** | 向导式流程，默认配置合理 | 完成率 > 80%，步骤 ≤ 3步 |
| **导入词条** | 支持JSON/YAML拖拽上传，实时预览 | 成功率 > 90%，支持1000+条批量导入 |
| **首次发布** | 一键发布，自动生成CDN链接 | 从创建到发布 < 5分钟 |

**关键设计决策**：
- 项目创建时默认包含 `zh-CN` 和 `en-US`
- 导入时自动识别命名空间（按文件路径或key前缀）
- 首次发布跳过复杂审批，直接发布

### 4.3 流程2：译员日常工作

| 环节 | 体验标准 | 验收指标 |
|------|----------|----------|
| **任务查看** | 清晰的待办列表，优先级明确 | 找到待翻译词条 ≤ 3次点击 |
| **翻译编辑** | 原文译文对照，实时保存 | 编辑响应 < 200ms，自动保存无感知 |
| **提交审核** | 一键提交，状态即时反馈 | 提交成功率 > 98%，反馈延迟 < 1秒 |
| **审核反馈** | 退回原因清晰，修改入口明确 | 修改完成率 > 85% |

**关键设计决策**：
- 工作台默认显示"我的待办"（待翻译 + 被退回）
- 编辑器采用左右对照布局（左原文 / 右译文）
- 提交时自动运行基础校验（ICU语法、占位符）

### 4.4 流程3：审核员工作

| 环节 | 体验标准 | 验收指标 |
|------|----------|----------|
| **任务查看** | 清晰的待审列表，来源信息完整 | 找到待审任务 ≤ 2次点击 |
| **对比检查** | 原文译文并排，差异高亮 | 理解变更内容 ≤ 5秒 |
| **审核操作** | 一键通过/退回，原因可选 | 单条审核耗时 < 30秒 |
| **批量审核** | 支持多选批量操作 | 批量操作成功率 > 95% |

### 4.5 流程4：发布管理员操作

| 环节 | 体验标准 | 验收指标 |
|------|----------|----------|
| **创建发布** | 变更预览清晰，范围可选 | 理解变更内容 ≤ 10秒 |
| **门禁检查** | 问题分级（阻断/警告），定位准确 | 检查完成 < 3秒，误报率 < 5% |
| **正式发布** | 一键确认，进度可见 | 发布耗时 < 30秒，成功率 > 99% |
| **获取链接** | CDN地址一键复制，多格式支持 | 获取链接 ≤ 2次点击 |

**关键设计决策**：
- 发布前强制展示变更摘要（新增/更新/删除统计）
- 阻断性问题必须修复，警告可跳过
- 发布后自动生成版本化的CDN链接

## 六、组件使用规范

### 6.1 按钮层级

| 类型 | 使用场景 | 视觉样式 |
|------|----------|----------|
| **主要按钮** | 页面主操作（创建、保存、提交） | Primary，实心 |
| **次要按钮** | 辅助操作（编辑、查看） | Default，描边 |
| **文字按钮** | 低频操作（删除、更多） | Text，无框 |
| **图标按钮** | 紧凑空间（表格内操作） | Icon only |

**按钮顺序**（从右到左）：
```
[ 取消 ] [ 保存 ]    ← 确认类操作
[ 次要 ] [ 主要 ]    ← 通用场景
```

### 6.2 表单规范

**字段宽度**：
- 短字段（名称、编码）：200px
- 中字段（描述、路径）：400px
- 长字段（内容、JSON）：100%（自适应）

**字段间距**：24px（垂直）

**验证反馈**：
- 实时验证：失焦时触发
- 提交验证：点击提交时全量验证
- 错误提示：字段下方红色文字

### 6.3 表格规范

**默认列宽分配**：
```
| 选择框 (48px) | 名称 (flex: 2) | 状态 (120px) | 时间 (180px) | 操作 (150px) |
```

**行操作**：
- 1-2 个操作：直接显示图标按钮
- 3+ 个操作：使用"更多"下拉菜单

**批量操作**：
- 选中行后，表格顶部显示批量操作栏
- 显示选中数量：[已选择 5 项] [批量通过] [批量退回]

## 七、状态与反馈

| 状态 | 徽章 | 说明 |
|------|------|------|
| PENDING | 🔴 待翻译 | 等待翻译 |
| TRANSLATING | 🟡 翻译中 | 机器翻译进行中 |
| REVIEWING | 🟠 审核中 | 等待审核 |
| APPROVED | 🟢 已通过 | 审核通过，待发布 |
| PUBLISHED | 🔵 已发布 | 已发布到CDN |

### 7.3 反馈机制

| 场景 | 反馈方式 | 持续时间 |
|------|----------|----------|

| 保存中 | 按钮 Loading 状态 | 直到完成 |
| 加载中 | 骨架屏 / Spin | 直到完成 |
| 异步任务 | 进度条 / 通知中心 | 实时更新 |

| 术语 | 说明 |
|------|------|
| Key | 文案唯一标识，如 `common.button.submit` |
| Namespace | 功能或模块维度的逻辑分组 |
| Locale | 语言及区域变体，如 `zh-CN`、`en-US` |
| ICU | ICU MessageFormat（复数、性别、选择等格式）|
| Release | 一次发布的版本，包含特定范围的翻译 |
