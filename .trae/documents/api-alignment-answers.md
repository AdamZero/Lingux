## 一、通用规范问题

### 1. 响应格式规范

**后端确认**: **选项 A** - 标准包装格式

**实际格式**:
```json
{
  "code": 0,           // 0 表示成功，非 0 表示错误
  "message": "success", // 错误时的提示信息
  "data": {}           // 响应数据
}
```

**说明**: 
- 当前 NestJS 项目使用全局异常过滤器 `HttpExceptionFilter`
- 成功响应需要手动包装，建议创建统一响应拦截器
- 错误响应由 `HttpExceptionFilter` 统一处理
