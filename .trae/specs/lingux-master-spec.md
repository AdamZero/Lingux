## 1. 文档概述

### 1.1 编写目的

本文档是 Lingux 多语言翻译中台的**唯一权威规格文档**，旨在：

- 统一团队对产品愿景、架构和实现路径的认知
- 作为设计、开发、测试、验收的基准依据
- 指导产品从 MVP 到企业级的演进过程

### 1.2 目标读者

- 产品经理：理解产品全貌和演进路线
- 架构师：掌握系统架构和技术选型
- 开发人员：了解功能规格和实现要求
- 测试人员：明确验收标准和测试范围
- 运维人员：知晓部署和监控方案

### 1.3 术语表

| 术语           | 定义                                        |
| -------------- | ------------------------------------------- |
| Key            | 文案唯一标识，如 `common.button.submit`     |
| Translation    | 某 Key 在某 Locale 的翻译条目               |
| Namespace      | 功能或模块维度的逻辑分组                    |
| Locale         | 语言及区域变体，如 `zh-CN`、`en-US`         |
| ICU            | ICU MessageFormat（复数、性别、选择等格式） |
| TM             | Translation Memory，翻译记忆库              |
| Release        | 一次发布的版本，包含特定范围的翻译          |
| ReleaseSession | 发布审批会话，管理发布流程                  |

## 2. 产品定位与价值主张

### 2.1 产品定位

**Lingux** 是一个面向**个人开发者、中小企业、大型企业**的多语言翻译中台，提供从词条管理、翻译工作流、质量门禁到发布回滚的**一站式解决方案**。

### 2.2 核心价值主张

| 价值维度     | 具体价值                                 |
| ------------ | ---------------------------------------- |
| **效率提升** | LLM 初译覆盖 70%+，人工处理时长下降 30%+ |
| **质量保障** | 门禁拦截率 99%+，翻译覆盖率 95%+         |
| **协作顺畅** | 深度集成企业协作平台（飞书/钉钉/企微）   |
| **可追溯**   | 完整审计日志，支持一键回滚               |
| **成本可控** | 翻译成本可视化，预算预警                 |

### 2.3 目标用户画像

| 用户类型   | 角色       | 核心痛点               | 核心需求                     |
| ---------- | ---------- | ---------------------- | ---------------------------- |
| 个人开发者 | Owner      | 手动维护翻译文件易出错 | 简单高效的翻译管理           |
| 中小企业   | PM/Dev     | 多语言版本同步困难     | 协作翻译和自动发布           |
| 大型企业   | 本地化团队 | 质量难以统一，成本高   | 质量门禁、成本管控、合规审计 |

## 5. 用户旅程与体验设计

### 5.1 用户生命周期

```
获客期 → 激活期 → 留存期 → 变现期 → 推荐期
```

### 5.2 核心用户旅程

#### 旅程 1: 新用户首次体验

```
访问官网 → 了解价值 → OAuth登录 → 创建项目 → 导入数据 → 完成首次发布
```

**关键体验点**:

- 官网清晰传达价值主张
- OAuth 一键登录，无密码负担
- 项目创建向导，降低决策成本
- 导入预览，避免数据错误

#### 旅程 2: 译员日常工作

```
查看任务 → 编辑翻译 → 提交审核 → 查看反馈 → 修改完成
```

**关键体验点**:

- 任务列表清晰，优先级明确
- 编辑器智能提示（TM/术语/ICU）
- 一键提交，状态实时同步
- 反馈闭环，修改可追溯

#### 旅程 3: 审核员审批流程

```
收到通知 → 查看待审 → 对比检查 → 通过/退回 → 完成审核
```

**关键体验点**:

- 多渠道通知（站内/企业微信/钉钉）
- 源文译文对比视图
- 批量审核支持
- 拒绝必须填写原因

#### 旅程 4: 发布管理员操作

```
创建发布 → 门禁检查 → 审批流程 → 正式发布 → 监控状态
```

**关键体验点**:

- 发布预览，变更一目了然
- 门禁报告，问题清晰定位
- 一键回滚，故障快速恢复
- 发布报告，便于复盘

### 5.3 体验设计原则

1. **减少认知负担**: 清晰的导航，明确的操作反馈
2. **提升操作效率**: 批量操作，快捷键支持，智能推荐
3. **保证数据安全**: 操作确认，版本回溯，审计日志
4. **促进团队协作**: 实时同步，评论沟通，任务分配

### 6.2 详细功能规格

#### 6.2.1 用户与权限模块

> **详细规格**: [modules/01-auth/spec.md](modules/01-auth/spec.md)

##### 6.2.1.1 认证体系

**FR-AUTH-001**: OAuth2 登录

- **功能描述**: 支持飞书、钉钉、企业微信 OAuth2 授权登录
- **详细规则**:
  - 首次登录自动创建用户账号，username 使用平台提供的唯一标识
  - 自动绑定企业信息（如果 OAuth 返回企业数据）
  - 支持多个 OAuth 账号绑定到同一用户
  - Token 有效期：Access Token 2小时，Refresh Token 7天
- **异常处理**:
  - OAuth 授权失败：返回友好错误页面，提示重试
  - 企业未授权应用：提示联系管理员开通

**FR-AUTH-002**: JWT 认证

- **功能描述**: 基于 JWT 的无状态认证
- **详细规则**:
  - Token 包含：userId, role, enterpriseId, exp
  - 每次请求携带 Authorization: Bearer <token>
  - Token 过期前 30 分钟自动刷新
- **安全要求**:
  - JWT Secret 定期轮换
  - 支持 Token 吊销（黑名单机制）

##### 6.2.1.2 权限体系

**FR-AUTH-003**: RBAC 权限控制

- **角色定义**:
  | 角色 | 权限范围 |
  |------|----------|
  | ADMIN | 项目所有操作，成员管理，发布管理 |
  | EDITOR | Key CRUD，翻译编辑，提交审核 |
  | REVIEWER | 审核翻译，查看所有内容 |
  | VIEWER | 只读访问 |

- **权限粒度**:
  - 项目级：是否可访问项目
  - 命名空间级：是否可操作特定命名空间
  - 语言级：是否可编辑特定语言

- **权限判定逻辑**:
  ```
  1. 检查用户是否有项目访问权限
  2. 检查用户角色是否允许该操作
  3. 检查命名空间权限（如适用）
  4. 检查语言权限（如适用）
  ```

**FR-AUTH-004**: 企业组织架构

- **功能描述**: 同步企业组织架构，实现基于部门的权限控制
- **详细规则**:
  - 支持 SCIM 2.0 协议同步
  - 支持飞书/钉钉/企微组织架构导入
  - 部门与项目关联，部门成员自动获得项目权限
  - 支持跨部门项目协作

#### 6.2.2 翻译工作流模块

##### 6.2.2.1 状态机设计

**FR-TRANS-002**: 状态流转规则

- **PENDING → REVIEWING**:

- **REVIEWING → APPROVED**:
  - 条件：审核人有权限
  - 操作人：REVIEWER 及以上
  - 副作用：记录审核人、审核时间

- **REVIEWING → PENDING**:
  - 条件：必须填写退回原因
  - 操作人：REVIEWER 及以上
  - 副作用：记录退回原因，通知译员

- **APPROVED → PUBLISHED**:
  - 条件：通过质量门禁
  - 操作人：ADMIN

```typescript
// 轮询分配逻辑
function assignReviewer(
  candidates: User[],
  pendingCounts: Map<string, number>,
): User {
  // 1. 过滤在线且未满负荷的审核人
  const available = candidates.filter(
    (u) => u.isOnline && pendingCounts.get(u.id) < u.maxWorkload,
  );

  // 2. 按当前待审任务数升序排序
  available.sort((a, b) => pendingCounts.get(a.id) - pendingCounts.get(b.id));

  // 3. 返回任务数最少的审核人
  return available[0];
}
```

```typescript
function calculateCost(
  provider: string,
  inputTokens: number,
  outputTokens: number,
): number {
  const pricing = {
    "openai-gpt4": { input: 0.03, output: 0.06 }, // $/1K tokens
    "openai-gpt35": { input: 0.0015, output: 0.002 },
    "claude-3": { input: 0.008, output: 0.024 },

}
```

##### 6.2.3.2 ICU 语法校验

**FR-QUALITY-002**: ICU MessageFormat 校验

**支持的 ICU 类型**:


5. 不支持 ICU 类型必须报错

**校验示例**:

```
✓ 合法: "Hello {name}, you have {count, plural, one {# message} other {# messages}}"
✗ 非法: "Hello {name"                    // 未闭合
✗ 非法: "You have {count, plural, one {1}}" // 缺少 other
✗ 非法: "Value: {123var}"                // 变量名非法
```

```typescript
function validatePlaceholders(
  source: string,
  target: string,
): ValidationResult {
  const sourcePlaceholders = extractPlaceholders(source);
  const targetPlaceholders = extractPlaceholders(target);

  const missing = sourcePlaceholders.filter(
    (p) => !targetPlaceholders.includes(p),
  );
  const extra = targetPlaceholders.filter(
    (p) => !sourcePlaceholders.includes(p),
  );

  return {
    valid: missing.length === 0 && extra.length === 0,
    missing,
    extra,
  };
}
```

**FR-TM-002**: 相似度匹配算法

**匹配类型**:

|--------|------|----------|----------|

**匹配算法**:

```typescript
function calculateSimilarity(source: string, tmSource: string): number {
  // 1. 标准化处理（去除多余空格、统一大小写）
  const normalized1 = normalize(source);
  const normalized2 = normalize(tmSource);

  // 2. 完全匹配检查
  if (normalized1 === normalized2) return 100;

  // 3. 编辑距离计算 (Levenshtein Distance)
  const distance = levenshteinDistance(normalized1, normalized2);
  const maxLength = Math.max(normalized1.length, normalized2.length);
  const similarity = (1 - distance / maxLength) * 100;

  // 4. 占位符差异惩罚
  const placeholderDiff = comparePlaceholders(source, tmSource);
  const adjustedSimilarity = similarity * (1 - placeholderDiff * 0.1);

  return Math.round(adjustedSimilarity);
}
```
