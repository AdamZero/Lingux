## 产品概览

### 产品定位

**Lingux** 是一个面向个人开发者、中小企业、大型企业的多语言翻译中台，提供从词条管理、翻译工作流、质量门禁到发布回滚的一站式解决方案。

### 核心价值主张

| 价值维度     | 具体价值                                 |
| ------------ | ---------------------------------------- |
| **效率提升** | LLM 初译覆盖 70%+，人工处理时长下降 30%+ |
| **质量保障** | 门禁拦截率 99%+，翻译覆盖率 95%+         |
| **协作顺畅** | 深度集成企业协作平台（飞书/钉钉/企微）   |
| **可追溯**   | 完整审计日志，支持一键回滚               |
| **成本可控** | 翻译成本可视化，预算预警                 |

### 核心工作流

```
录入 (Key) → 翻译 (Translation) → 审核 (Review) → 发布 (Release) → 回滚 (Rollback)
```

### 当前进度

| 状态      | 数量 | 说明                                                          |
| --------- | ---- | ------------------------------------------------------------- |
| ✅ 已完成 | 18项 | 用户认证、项目管理、命名空间、词条管理、翻译工作流、导入/导出 |
| 🔄 进行中 | 4项  | 发布审批流程前端、搜索与筛选                                  |

## 术语表

| 术语               | 定义                                        |
| ------------------ | ------------------------------------------- |
| **Key**            | 文案唯一标识，如 `common.button.submit`     |
| **Translation**    | 某 Key 在某 Locale 的翻译条目               |
| **Namespace**      | 功能或模块维度的逻辑分组                    |
| **Locale**         | 语言及区域变体，如 `zh-CN`、`en-US`         |
| **ICU**            | ICU MessageFormat（复数、性别、选择等格式） |
| **TM**             | Translation Memory，翻译记忆库              |
| **Release**        | 一次发布的版本，包含特定范围的翻译          |
| **ReleaseSession** | 发布审批会话，管理发布流程                  |

## 贡献指南

### 如何更新规格

1. 确定变更范围（单个模块 / 多个模块 / 全局变更）
2. 修改对应文档
3. 更新版本号和最后更新时间
4. 如有重大变更，同步更新本文档状态

### 文档 Review 流程

1. 创建文档变更 PR
2. 相关角色 Review（产品/架构/开发）
3. 通过后合并
4. 更新本文档状态
