# 多语言管理平台 PRD（功能版，不涉技术实现）

文档版本：v0.1（功能对齐稿）  
适用阶段：功能评审/范围确认/排期拆分  
注意：本稿聚焦“能做什么”，不讨论数据库/接口/技术栈

## 1. 背景与目标
- 背景：产品多端、多区域发布，对文案收集、翻译与更新效率要求高；需兼容企业内常用协同平台（飞书/钉钉/企业微信）的人事与审批流。
- 目标：
  - 将多语言资产集中化管理，降低漏翻、错翻与回归成本。
  - 用 LLM 提升初译效率，以术语/校验/审批保障质量与一致性。
  - 以“项目-命名空间-环境”为单位，支持灰度发布与快速回滚。
  - 深度融入企业协作生态（登录、通知、审批），降低使用门槛。
- 成功标准（功能视角）：
  - 翻译覆盖率≥95%，ICU/占位符校验拦截率≥99%。
  - 人工审核通过前，平台阻断未合规译文发布。
  - 接入任意一种企业协作平台实现免登+待办通知。
  - LLM 初译覆盖≥70% 新增条目；人工平均处理时长下降≥30%。

## 2. 范围与非范围
- 范围内：
  - 多语言资产管理（项目/命名空间/环境/键/条目/版本）。
  - 翻译工作流（机器初译→人工校对→审批→发布）。
  - LLM 与第三方翻译服务接入；术语/翻译记忆（TM）。
  - 质量门禁（ICU、占位符、术语、敏感词、长度/像素）。
  - 企业协作平台接入（免登/通知/审批卡片）。
  - 导入/导出；发布与回滚；监控与成本统计；RBAC 与审计。
- 非范围（当前阶段）：
  - 端侧 SDK/CLI 的技术细节与发布机制。
  - 全量流程编排引擎与自定义脚本能力。
  - 复杂 BI 报表定制（保留基础指标与可导出能力）。
  - 在线实时冲突检测与手动合并流程（保留导入/在线编辑的基础冲突处理）。
  - 复杂版本对比与多版本注释（保留基础版本回溯与回滚）。
  - 灰度发布策略与多阶段流量编排。
  - 回译差异校验与相关评分流程。

## 3. 用户与角色
- 角色：
  - Owner/管理员：全局配置、项目创建、发布策略、权限管理。
  - 产品/PM：文案需求发起、优先级与发布节奏管理、审批。
  - 开发/Dev：新增键、更新源文案、导入导出、集成管道。
  - 译员/Translator：处理指定语言条目、遵循术语校对。
  - 审核/Reviewer：复核译文、退回修改或通过。
  - 审计/Auditor：审计日志与合规导出、只读。
  - 游客/Guest：只读访问指定项目或环境。
- 登录与组织：
  - 支持飞书/钉钉/企业微信免登；按组织/部门同步角色。

## 4. 核心原则
- 质量优先：任何发布前必须通过门禁。
- 上下文驱动：截图/模块/变量示例辅助翻译质量。
- 可追溯：所有变更具备版本链路、可回滚。
- 成本可控：LLM/服务调用可限额、可统计、可审计。
- 最小必要权限：按资源粒度（项目/命名空间/语言）授权。

## 5. 术语与约定
- Key：文案唯一标识。
- Entry：某 Key 在某 Locale 的翻译条目。
- Namespace：功能或模块维度的逻辑分组。
- Locale：语言及区域变体（如 zh-CN、pt-BR）。
- ICU：ICU MessageFormat（复数、性别、选择等格式）。
- 环境：dev/stg/prod 等发布目标。
- TM（Translation Memory）：翻译记忆库。
- 术语库：标准术语与禁用词集合。

## 6. 功能需求

### 6.1 项目/命名空间/环境
- 用户故事
  - 管理员可创建项目，配置默认语言与可用环境。
  - Dev/PM 将键按命名空间维护，避免跨模块污染。
- 功能点
  - 项目创建/编辑；命名空间增删改；环境开启/关闭。
  - 标签/业务属性：为项目/命名空间设置标签用于筛选。
  - 只读环境：如 prod 仅允许发布、更改需走流程。
- 验收
  - 能创建多个环境并可按环境发布。
  - 命名空间可用于筛选、导出、发布打包。

### 6.2 键/条目与上下文
- 用户故事
  - Dev 新增键时定义 ICU 与占位符，并上传截图/备注。
  - 译员根据上下文（截图、UI路径、变量示例）进行翻译。
  - 译员需要按网站页面维度查看与处理文案（如 /about 页）。
  - Dev/PM 为静态资源（图片 alt、下载文件名）配置可翻译文案。
- 功能点
  - 键管理：新增/编辑/停用；描述与标签；占位符定义。
  - 文本类型：纯文本/Markdown/HTML；ICU 作为一等能力。
  - 上下文：截图绑定、UI 路径、字符/像素限制提示。
  - 页面与文案关联：为 Key/条目绑定网站页面路径（如 /home、/about）与页面标题；支持按页面路径筛选与批量查看。
  - 静态资源翻译支持：为图片/图标/下载文件等资源的 alt/说明/文件名绑定 Key；可记录资源路径，导出时携带。
  - 源文案变更联动（简化）：源文案变更后，系统自动将所有目标语言标记为“过期”，在列表中高亮；支持一键批量触发机器初译。
- 验收
  - 占位符在源/译文对齐校验通过。
  - ICU 解析通过；富文本保真提示。
  - 可为条目绑定页面路径并按路径筛选，结果准确。
  - 可为静态资源绑定 Key 并翻译；导出包含资源路径。

### 6.3 翻译工作流（可配置）
- 用户故事
  - PM 发起批量翻译任务；LLM 初译；译员修改；Reviewer 审核；审批后进入发布。
- 功能点
  - 工作流状态：draft → machine_translated → in_review → approved → published。
  - 任务拆分：按语言/命名空间/标签；支持批量领取/指派/到期提醒。
  - 评论与讨论：对条目/版本级别评论，@人与历史可追溯。
- 验收
  - 任意状态变更均可审计；退回需填写原因。
  - 支持批量就绪（全选通过质量门禁的项）。

### 6.4 LLM/翻译服务接入与提示管理
- 用户故事
  - PM 为项目选择模型与提示模板；控制成本与速率。
- 功能点
  - Provider 管理：添加/启用/禁用；预算、QPS 限制。
  - 提示模板：按项目/语言/领域配置；注入术语与上下文。
  - 回退策略：主服务失败自动切换备选；缓存复用结果。
- 验收
  - 可在任务中一键机器初译；失败有明确信息与重试机制。
  - 成本按项目/语言可视化。

### 6.5 术语库与翻译记忆（TM）
- 用户故事
  - 译员在编辑时看到术语高亮与 TM 建议。
- 功能点
  - 术语：术语定义、目标语对应、禁用词、备注，编辑审批。
  - TM：导入历史翻译；相似度匹配；一键采纳。
- 验收
  - 术语违规拦截；TM 建议可统计采纳率。

### 6.6 质量门禁与校验
- 用户故事
  - 发布前系统自动跑校验；未达标阻断。
- 功能点
  - 校验项：占位符匹配、ICU 语法、术语一致性、敏感词（支持行业词库）、长度/像素限制、静态资源文案长度限制。
  - 合规与敏感词：按行业配置敏感词库；出现敏感词时阻断并给出替换建议；支持在条目中添加合规备注。
  - 策略：按项目/环境配置“阻断/警告”；可设最小覆盖率。
- 验收
  - 门禁报告可定位到具体条目与失败原因。
  - 设置不同环境门槛（如 prod 严格、dev 宽松）。

### 6.7 导入/导出
- 用户故事
  - Dev 将遗留资源导入；PM 导出给内部译员或备份。
- 功能点
  - 格式：i18next JSON、YAML、XLIFF、PO、ARB、CSV/Excel。
  - 策略：冲突处理（覆盖/跳过/新版本）；保留注释与上下文。
  - 字段扩展：导出可包含页面路径与资源路径字段，便于对接网站。
- 验收
  - 导入后键/条目状态合理；导出可按项目/命名空间/语言筛选。

### 6.8 发布与回滚
- 用户故事
  - PM 将已批准的翻译打包为发布；故障时快速回滚。
- 功能点
  - 发布：选择目标环境与命名空间；可按页面路径批量发布（仅发布指定页面相关文案）；生成资源包；记录版本号与说明。
  - 回滚：支持按页面路径或版本一键回退；发布历史可视化。
  - 发布报告：简要展示涉及页面、语言与版本信息，便于复盘。
- 验收
  - 发布前自动门禁；发布产物可按版本追溯。
  - 回滚不改变审计历史，仅改变环境已生效版本指针。
  - 页面维度的发布/回滚操作可独立进行且不影响其他页面。

### 6.9 企业协作平台集成（飞书/钉钉/企业微信）
- 用户故事
  - 用户免登进入平台；审核/发布在平台内收到卡片并操作。
- 功能点
  - 免登：平台身份绑定内部账号。
  - 审批卡片：条目/发布审批，支持同意/退回/评论。
  - 通知：待办提醒、逾期提醒、发布完成通知；可订阅。
- 验收
  - 至少一个平台完成免登与审批卡片闭环。
  - 操作结果在平台与系统双向同步。

### 6.10 权限与审计
- 用户故事
  - 管理员配置不同角色权限；审计导出合规记录。
- 功能点
  - RBAC：按项目/命名空间/语言授予操作权限；字段级保护（源文案/占位符不可随意改）。
  - 审计日志：变更记录、IP、时间、对象、前后差异；可导出。
- 验收
  - 未授权用户无法访问或修改受限资源。
  - 审计导出覆盖所有敏感操作。

### 6.11 监控与报表
- 用户故事
  - PM 查看覆盖率、滞后、成本；识别高风险模块。
- 功能点
  - 指标：翻译覆盖率、审核通过率、逾期任务、门禁失败率、TM/术语采纳率、成本。
  - 视图：保存过滤条件生成看板；支持订阅周报。
- 验收
  - 指标计算规则明确；看板刷新与导出可用。

### 6.12 可用性与易用性
- 功能点
  - 所见即所得编辑器（字符/像素计数提示）。
  - 高级搜索（键、状态、语言、命名空间、标签、负责人）。
  - 批处理（批量状态变更、批量指派）。
- 验收
  - 常见过滤保存为视图；编辑体验响应流畅。

## 7. 关键工作流（文字版）
- 文案新增：Dev 新增键 → 自动初译 → 校验 → 指派译员 → 译员修改 → Reviewer 审核 → 审批卡片通过 → 进入发布候选。
- 回归更新：源文案变更 → 标记受影响语言为“过期” → 触发任务 → 初译与对齐 → 审核 → 合并。
- 发布门禁：对目标环境运行门禁 → 全绿 → 生成版本 → 按页面路径/命名空间发布 → 通知；必要时按页面一键回滚。

## 8. 权限规则（摘要）
- Dev/PM 可新增/修改源文案与占位符；译员不可改源文案与占位符。
- 译员仅可编辑被指派语言条目；Reviewer 可审核通过/退回。
- 发布与回滚仅限 Admin/PM。
- 术语修改需管理员或具备“术语管理员”权限的角色审核通过。

## 9. 非功能性（功能视角下的期望）
- 可用性：核心操作（保存译文、创建任务、发布）在工作时间可靠。
- 性能：常用列表/详情的交互延迟可接受（用户感知顺畅）。
- 安全：组织隔离、最小权限、密钥与敏感数据不在页面泄露。
- 可观测：关键操作有日志与基础监控指标；错误可定位。

## 10. MVP 范围与阶段计划（功能切片）
- 阶段 1（MVP）
  - 项目/命名空间/环境；键/条目管理与上下文（ICU/占位符）。
  - 页面与文案关联；静态资源文案支持（alt/说明/文件名，含资源路径）。
  - 工作流：机器初译（接 1–2 个服务）→ 人工审核 → 发布。
  - 门禁：占位符/ICU/未翻译；基础术语提示与行业敏感词拦截。
  - 企业协作：至少一种平台免登 + 待办通知。
  - 导入/导出：JSON/i18next + CSV/Excel。
  - 发布与回滚：支持页面维度发布与回滚；版本化与一键回退。
  - 权限与审计：基础 RBAC + 审计日志。
  - 监控：覆盖率、门禁失败、成本基础统计。
- 阶段 2
  - 审批卡片闭环；术语库完善与强校验；TM 匹配与采纳。
  - 截图上下文增强；成本/速率更细管理。
- 阶段 3
  - A/B 质量评估；多租户运营；SCIM 同步；移动 OTA 发布视角。

## 11. 度量指标（功能验收口径）
- 质量：门禁失败率、术语/敏感词违规次数、静态资源长度违规次数。
- 效率：从“键创建”到“发布”的平均周期；译员吞吐（条/天）。
- 覆盖：多语言覆盖率与未翻译 Top 列表。
- 成本：按项目/语言/服务商的月度费用与单条成本。
- 使用：审批响应时长、通知到达率、看板订阅使用率。

## 12. 风险与依赖
- 风险
  - LLM 质量波动：需以术语/门禁缓解；保留人工兜底。
  - 平台集成策略变更（飞书/钉钉/企微）：要求接口演进适配与灰度。
  - ICU/占位符误用导致高失败：需提供友好提示与示例。
- 依赖
  - 企业组织与账号体系连通（免登与人员同步）。
  - 术语/历史翻译素材的导入配合。
  - 发布目标环境（CDN/制品仓库/应用侧拉取）准备度。

## 13. 非目标与边界
- 不作为终端 UI 翻译的唯一入口（允许外包离线处理后导入）。
- 不提供高级排版/富设计能力（仅支持必要的富文本校验与提示）。
- 不负责业务端的动态布局适配（可提供长度/像素提示与警告）。

## 14. 开放问题（功能层面待定）
- 审批卡片的层级：是否需要“项目级发布审批”与“条目变更审批”并行？
- 术语变更是否强制影响历史译文（批量替换）还是仅对新条目生效？
- 组织同步节奏：实时或定时拉取？外部译员归属与数据可见范围。
- 门禁阈值默认策略如何按环境预置（例如 prod 严格，dev 宽松）？
- 公开导出是否需要水印/脱敏策略（防数据泄露）？
