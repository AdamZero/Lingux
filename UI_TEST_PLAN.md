# Lingux 多语言翻译中台 - UI 测试计划

**文档版本**: v1.0  
**最后更新**: 2026-03-17  
**测试工具**: Chrome DevTools MCP + webapp-testing skill

---

## 1. 测试概述

### 1.1 测试目标

验证 Lingux 前端应用的各项功能是否符合产品规格，确保用户体验流畅、功能完整。

### 1.2 测试范围

| 模块 | 测试内容 | 优先级 |
|------|----------|--------|
| 用户认证 | OAuth 登录、JWT 认证、权限控制 | P0 |
| 项目管理 | 项目 CRUD、语言配置、命名空间 | P0 |
| 词条管理 | Key CRUD、标签管理、筛选搜索 | P0 |
| 翻译工作流 | 翻译编辑、状态流转、审核流程 | P0 |
| 发布管理 | 发布创建、审批、回滚 | P0 |
| 工作台 | 统计展示、快捷操作、待办任务 | P1 |
| 响应式布局 | 桌面端、平板端、移动端适配 | P1 |
| 性能测试 | 页面加载速度、交互响应 | P2 |

### 1.3 测试环境

- **前端**: http://localhost:5175 (Vite + React + Antd)
- **后端**: http://localhost:3001 (NestJS)
- **数据库**: PostgreSQL
- **浏览器**: Chrome (通过 DevTools MCP)

### 2.7 性能测试 (PERF)

#### PERF-001: 页面加载性能
**测试步骤**:
1. 使用 Chrome DevTools Performance
2. 记录页面加载过程

**预期结果**:
- [ ] LCP < 2.5s
- [ ] CLS < 0.1
- [ ] TTFB < 600ms

**实际结果**:
- LCP: 1,918 ms (⚠️ 接近阈值)
- CLS: 0.03 (✅ 良好)
- TTFB: 11 ms (✅ 优秀)

**状态**: ⚠️ 需优化

**优化建议**:
- 优化 React 组件渲染性能
- 使用代码分割减少初始加载
- 延迟加载非关键组件

## 5. 测试建议

### 5.1 短期建议

1. **修复 404 问题**: 添加 `/translations` 路由或修改工作台"新建翻译"按钮行为
2. **统一环境配置**: 确保前后端 FRONTEND_URL 配置一致
3. **完善测试数据**: 创建测试项目和词条，覆盖更多测试场景

### 5.2 中期建议

1. **自动化测试**: 使用 Playwright 编写自动化测试脚本
2. **视觉回归测试**: 使用 Percy 或 Chromatic 进行 UI 对比
3. **性能监控**: 集成 Lighthouse CI 进行性能监控

### 5.3 长期建议

1. **E2E 测试覆盖**: 覆盖完整的用户旅程（创建项目 → 添加 Key → 翻译 → 审核 → 发布）
2. **跨浏览器测试**: 测试 Chrome、Firefox、Safari 兼容性
3. **移动端测试**: 使用真实移动设备进行测试
